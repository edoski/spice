RPC Acquisition

RPC acquisition is the concrete block source that talks to Ethereum-compatible JSON-RPC endpoints through Web3. It enriches block headers with fee-history and priority-fee information needed by later features.

src/spice/acquisition/rpc Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

It implements the acquisition source interface used by spice.acquisition.pull and feeds corpus assembly.

Beginner Mental Model

This module is the network adapter. The generic puller asks for block ranges; the RPC client translates that into batched JSON-RPC calls, handles provider retry behavior, and returns typed block rows.

Main Source Files

client.py
Defines block header parsing, fee-history enrichment, supported enrichment flags, and BlockRpcClient.
transport.py
Builds retrying async Web3 transports with project user-agent and retry configuration.
__init__.py
Exports the RPC acquisition package boundary.

Input And Output Flow

  • Build an async Web3 transport for a provider endpoint.
  • Fetch headers and fee history for the requested range.
  • Build canonical row data for each block.
  • Let the generic pull controller handle scheduling and validation.

Common Confusing Concepts

Fee history enrichment

Headers alone are not enough for fee modeling. Fee history calls supply reward percentile data and base-fee context.

Batch transport

Many RPC calls are grouped to reduce network overhead, but the adaptive controller still protects providers from oversized requests.
