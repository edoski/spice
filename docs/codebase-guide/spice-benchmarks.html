Spice beginner guide

Benchmarks

Benchmarks materialize experiment grids into durable workflow plans, submit them, collect evaluation results, and maintain/export result indexes.

src/spice/benchmarks Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

Benchmarks coordinate config, workflows, execution, storage, and evaluation. They do not own model math or storage root internals.

Beginner Mental Model

A benchmark is a controlled set of workflow runs. The plan says what to run; submission starts jobs; collection finds matching evaluation summaries; the index makes results easy to compare.

__init__.py
Benchmark package boundary.

Input And Output Flow

  • A benchmark YAML spec is loaded.
  • Plan materialization expands dimensions into workflow entries.
  • Submission sends entries locally or remotely with dependencies.
  • Collection resolves produced evaluations and writes index rows.

Common Confusing Concepts

Plan vs run

The plan is what should happen. A run is a concrete materialized instance with metadata and submissions.

Dimension

One axis of variation, such as model, delay, or problem choice.

Collection resolver

The matching layer that decides which produced evaluation summary belongs to which planned benchmark case.
