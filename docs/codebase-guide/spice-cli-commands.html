Spice beginner guide

CLI Commands

This package holds concrete command handlers for workflows, config inspection, storage inspection/deletion, transfers, and benchmark operations.

src/spice/cli/commands Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

It sits under the CLI app and above all domain services. Each file groups commands by operator task.

Beginner Mental Model

Commands are adapters. They collect flags, build typed inputs, call the right service, and format the answer. They should not duplicate config resolution, storage selection, or workflow internals.

Main Source Files

benchmark.py
Plans, submits, collects, shows, rebuilds, lists, and exports benchmark runs and benchmark result indexes.
config.py
Lists config groups, shows raw config documents, and opens config files for editing.
storage.py
Implements show/delete commands for dataset, study, and artifact roots using selector objects and storage operator services.
transfer.py
Pushes dataset roots and pulls artifact roots through the execution transfer boundary.
workflows.py
Runs or submits acquire, train, tune, and evaluate workflows.
__init__.py
Keeps the command package importable.

Input And Output Flow

  • A command handler receives parsed options.
  • It builds a workflow selection, storage selector, or benchmark request.
  • It calls the domain service.
  • It renders structured output or reports submission progress.

Common Confusing Concepts

Selector rule

Commands accept user-friendly selectors, but storage code resolves them against catalog/root state.

Local vs remote run

The same command may run locally or submit a resolved snapshot remotely. Remote execution starts after config has already been resolved.
