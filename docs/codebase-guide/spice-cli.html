Spice beginner guide

CLI

The CLI is the operator edge. It owns Typer command registration, command defaults, shared options, output formatting, and conversion of domain errors into readable terminal messages.

src/spice/cli Docs: ARCHITECTURE.md

Where It Fits

It is above config, workflows, storage, execution, and benchmarks. It should pass explicit values downward instead of hiding CLI-only defaults in lower packages.

Beginner Mental Model

The CLI translates human commands into typed selections and workflow calls. It is allowed to be convenient. The deeper code should be deterministic and explicit.

Main Source Files

app.py
Registers the Typer app and command groups, then exposes the spice console entry point.
errors.py
Adapts operator-facing exceptions into Typer errors with consistent messages.
options.py
Defines shared command options and CLI defaults such as the default storage root and remote target.
output.py
Prints JSON, key-value rows, warnings, and sectioned command output.

Input And Output Flow

  • Typer parses command flags.
  • Command modules build typed selections or storage queries.
  • Config/workflow/storage/execution packages do the real work.
  • CLI output helpers render the result.

Common Confusing Concepts

Remote default boundary

DEFAULT_REMOTE_TARGET = "disi_l40" belongs here. Execution code receives a concrete target name and does not guess.

CLI vs workflows

CLI commands should not train models directly. They prepare arguments and call workflow or execution services.
