Spice beginner guide

Checked-In Config Specs

This package stores the YAML specs that users and workflows name. These files are the checked-in catalog of chains, datasets, providers, models, objectives, surfaces, benchmarks, and tuning spaces.

src/spice/conf Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

Config loading reads this directory. Runtime code should see typed models produced from these YAML files, not raw dictionaries.

Beginner Mental Model

If src/spice/config is the loader and validator, src/spice/conf is the recipe book. A surface recipe chooses many smaller recipes, and workflow resolution validates the final meal before anything runs.

Main Source Files

__init__.py
Marks the YAML tree as package data so the wheel can include src/spice/conf/**/*.yaml.
benchmark/*.yaml
Benchmark sweeps and named benchmark cases.
chain/*.yaml
Chain identity and chain-level facts.
dataset/*.yaml
Dataset names, dates, and history/evaluation windows.
dataset_builder/*.yaml
Dataset-builder implementation choices such as fixed sequence temporal windows.
evaluation/*.yaml
Evaluator specs such as Poisson replay and full temporal replay.
execution/*.yaml
Remote execution target specs.
features/*.yaml
Feature family selections.
model/*.yaml
Model family configs.
objective/*.yaml
Optimization objective specs.
prediction/*.yaml
Prediction family selections.
problem/*.yaml
Temporal problem compiler specs.
provider/*.yaml
RPC provider settings.
surface/*.yaml
Named workflow bundles.
training/*.yaml
Training loop settings.
tuning/*.yaml and tuning_space/*.yaml
Optuna run settings and search spaces.

Input And Output Flow

  • A workflow selection names a surface.
  • The surface names group specs such as model, problem, features, and objective.
  • Typed loading validates every YAML file against its owning model.

Common Confusing Concepts

YAML identity rule

The file name and the internal id must agree. This prevents a file path from lying about the thing inside it.

Declarative, not executable

YAML chooses behavior. Python compilers execute behavior after validation.
