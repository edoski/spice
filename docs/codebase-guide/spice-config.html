Spice beginner guide

Config

Config turns human-facing names, YAML fragments, workflow selections, and persisted snapshots into typed runtime config objects. It is where string references become validated objects.

src/spice/config Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

It sits between CLI/benchmarks and every runtime domain. Downstream workflow code should receive resolved config models instead of resolving names again.

Beginner Mental Model

A user says something small like surface current_row_fee_dynamics and model lstm. Config expands that into the exact chain, dataset, provider, features, problem, model, training, objective, evaluation, storage, and tuning objects needed by a workflow.

Main Source Files

group_catalog.py
Defines known config groups, their directories, identity rules, and validators shared by raw and typed loading.
groups.py
Loads YAML mappings from src/spice/conf for show/edit/template use and low-level payload access.
models.py
Defines common config models: workflow task, chain, dataset, storage, problem, split, early stopping, and helper coercion.
resolution.py
Freshly resolves acquire, train, tune, and evaluate selections into workflow config objects.
resolved_workflows.py
Assembles already resolved train, tune, and evaluate field sets and owns evaluate defaults.
selections.py
Defines unresolved workflow-selection models and metadata consumed by CLI and benchmark planning.
surfaces.py
Defines surface frame models. A surface is a named bundle of refs for common workflows.
typed_groups.py
Loads named groups into typed owner configs for runtime resolution.
workflow_snapshots.py
Serializes and hydrates already resolved workflow configs for remote execution and benchmark plans.
__init__.py
Exports the public config boundary used by other packages.

Input And Output Flow

  • A CLI command or benchmark creates a workflow selection.
  • The selected surface and overrides become workflow-shaped refs.
  • Named YAML groups are loaded and owner coercers rebuild concrete nested config types.
  • The result is an AcquireConfig, TrainConfig, TuneConfig, or EvaluateConfig.

Common Confusing Concepts

Surface

A named bundle of defaults. It is not the final config; it is a starting point that can be overridden.

Owner coercer

The package that owns an implementation knows how to validate its concrete config. Config delegates that detail instead of importing every field rule into one huge module.

Fresh resolution vs hydration

Fresh resolution starts from names. Hydration starts from a saved resolved payload and rebuilds typed objects without choosing names again.
