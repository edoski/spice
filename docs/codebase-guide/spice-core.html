Spice beginner guide

Core

Core holds small primitives that many packages need: strict config model behavior, operator-facing errors, safe file replacement, rendering helpers, reporting, path validation, and generic local-spec helpers.

src/spice/core Docs: ARCHITECTURE.md

Where It Fits

It is the lowest shared layer. Other packages may depend on core, but core must not know about the CLI, storage roots, blockchain details, model families, or evaluator choices.

Beginner Mental Model

Core is the toolbox, not the application. If a helper is generic enough that no domain should own it, it can live here. If it knows what a dataset, model, evaluator, or remote target is, it belongs elsewhere.

Main Source Files

async_runtime.py
Runs async entry points in a way that handles keyboard interruption cleanly for command code.
config_model.py
Provides the strict Pydantic base model used by concrete config models. Strict models reject surprise fields early.
constants.py
Stores shared file-name constants such as the model state file name.
errors.py
Defines operator-facing exception types for config, missing state, invalid layout, selector resolution, and conflicts.
files.py
Implements atomic writes, atomic path replacement, removal, and empty-directory pruning so storage operations avoid half-written state.
rendering.py
Turns typed values and metric summaries into readable command output fields.
reporting.py
Defines the Reporter callback wrapper used by workflows to emit progress without tying lower layers to Typer.
specs.py
Contains generic spec-table helpers: read ids from payloads, look up local specs, validate concrete config types, and normalize owner errors.
validation.py
Validates path segments so generated storage ids and operator-provided names cannot accidentally become paths.

Input And Output Flow

  • Domain package declares a local spec table.
  • core.specs validates payload ids and config classes.
  • Domain package compiles the concrete config into a local contract.

Common Confusing Concepts

Local spec

A local spec is a registry row inside the package that owns an implementation. It maps an id like lstm to config validation and compiler logic.

Atomic file writes

The code writes a temporary path first, then replaces the final path. Readers either see the old file or the new file, not a half-written file.
