Spice beginner guide

Corpus

Corpus turns acquired block data into a canonical, validated dataset with history/evaluation splits and metadata. It is the durable raw-data foundation for feature building and modeling.

src/spice/corpus Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

It sits after acquisition and before features, temporal problems, and modeling. Storage persists corpus roots, but corpus owns the block schema and coverage rules.

Beginner Mental Model

A corpus is the clean notebook of blockchain history. Before any model can learn, the system needs rows with stable columns, known date ranges, no accidental gaps, and enough history before evaluation windows.

Plans history refill and materialization needs based on temporal capability requirements.

Summarizes block frames, validates contiguous ranges, and reports exact window counts.

Undocumented child implementation that materializes history/evaluation parquet splits and their models.

Corpus package boundary.

Input And Output Flow

  • Acquisition stages raw canonical rows.
  • Corpus validation checks schema, ordering, and gaps.
  • Split materialization writes history/evaluation block files.
  • Metadata records identity, provider, coverage, and validation facts.

Common Confusing Concepts

Canonical row

A normalized block record with project-approved column names and types. Later code depends on this contract.

History vs evaluation split

History gives the model past context. Evaluation is the future window where decisions are scored.

Coverage

Coverage is not just row count. The rows must cover the right time span before and during the target window.
