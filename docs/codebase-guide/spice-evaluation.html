Spice beginner guide

Evaluation

Evaluation scores decoded model decisions against temporal fee outcomes. It owns evaluator configs, evaluator contracts, replay adapters, temporal accounting, metric descriptors, and evaluation summaries.

src/spice/evaluation Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

It sits after prediction decoding and is used by modeling scoring, train/tune objectives, and evaluate workflows.

Beginner Mental Model

Prediction says what action the model chose. Evaluation asks what would have happened if that action were used and summarizes the economic result.

Input And Output Flow

  • Modeling decodes predictions into a supported result id.
  • Evaluator validates it accepts that result type.
  • Temporal replay simulates or replays decision timing.
  • Metric summaries are returned and persisted.

Common Confusing Concepts

Decoded result boundary

Evaluators do not consume raw tensors. They consume typed decisions such as decoded offsets.

Poisson replay

A stochastic replay using sampled arrivals to approximate event timing.

Full temporal replay

A direct replay across the available temporal samples.

Metric descriptor

A stable description of a metric id, direction, and display meaning.
