Spice beginner guide

Features

Features convert canonical block rows into numeric columns a model can use. Feature contracts describe required source columns, produced feature names, and semantic fingerprints.

src/spice/features Docs: ARCHITECTURE.md

Where It Fits

Features sit between corpus rows and temporal problem stores. They do not choose samples or train models; they prepare the row-level measurements that later stages consume.

Beginner Mental Model

A model cannot learn from raw concepts like "recent fee pressure" unless those ideas become numbers. Features compute those numbers consistently and record what they mean.

Input And Output Flow

  • Validate selected feature names.
  • Build source series from canonical block rows.
  • Compute configured feature columns.
  • Return a resolved feature table plus feature semantics.

Common Confusing Concepts

Source vs feature

A source column comes from canonical data. A feature column is computed from source columns.

Fingerprint

A stable description of selected feature meaning. It helps compare artifacts and datasets later.

Feature family

A related group of feature definitions registered together, such as core fee dynamics.
