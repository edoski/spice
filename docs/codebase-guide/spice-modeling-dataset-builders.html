Dataset Builders

Dataset builders turn temporal problem stores and feature tables into tensors and sample selections used by training and inference.

src/spice/modeling/dataset_builders Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

They sit between temporal/features and model training. Modeling owns them because they produce model-ready data, but temporal contracts define the problem facts they consume.

Beginner Mental Model

A dataset builder is the packing step. It takes aligned rows and problem facts, chooses train/validation/inference samples, scales features, and packs fixed-shape arrays that PyTorch can process.

Main Source Files

base.py

fixed_sequence_temporal.py
Concrete builder that prepares fixed-length context sequences for training and inference.
preparation.py
Defines preparation contexts, sample selections, split indices, temporal-fact alignment checks, and prepared dataset objects.

Dataset-builder package boundary.

Input And Output Flow

  • Validate that required features exist.
  • Select sample positions for train/validation/evaluation windows.
  • Fit and apply input normalization.
  • Build fixed-length tensors and runtime metadata.

Common Confusing Concepts

Tensorization

Turning tables into numeric arrays with fixed dimensions so neural networks can process them.

Fixed sequence

Each sample gets the same number of context rows. Short or unavailable history must be handled explicitly.

Split indices

Train and validation are positions in the same prepared store, not separate concepts invented by the model family.
