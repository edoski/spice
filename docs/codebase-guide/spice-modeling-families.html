Spice beginner guide

Model Families

Model families define neural-network architectures, their configs, tuning spaces, tuned parameter types, and construction logic.

src/spice/modeling/families Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

They sit inside modeling and are selected by config. Prediction families define output contracts; model families build networks that produce the required output heads.

Beginner Mental Model

A model family is a blueprint for a neural network. The current blueprints process sequences of fee features and emit named heads that the prediction family can train and decode.

Main Source Files

base.py
Defines common model config, tuning-space config, tuned params, and tunable-field validation.
_heads.py
Builds reusable MLP and temporal output heads.
_sequence.py
Handles sequence lengths and selecting the last valid timestep.
_transformer_shared.py
Shares transformer positional encoding, encoder config, tuning validation, and encoder construction.
lstm.py
Concrete LSTM baseline family, config, tuning space, tuned params, and build function.
transformer.py
Concrete transformer baseline family and tuning behavior.
transformer_lstm.py
Hybrid transformer-plus-LSTM baseline family.
registry.py
Loads model specs, coerces configs, builds models, samples tuned params, and applies tuned params.
__init__.py
Model-family package boundary.

Input And Output Flow

  • Config selects a model id and hyperparameters.
  • Registry validates the concrete model config.
  • The model builder receives input/output specs.
  • Training runtime optimizes the resulting PyTorch module.

Common Confusing Concepts

Architecture vs prediction task

The architecture decides how features are processed. The prediction family decides what the outputs mean.

Head

A final network branch with a named output, such as offset logits or fee regression.

Tuning space vs tuned params

A tuning space lists candidates. Tuned params are one chosen set from an Optuna study.
