Spice beginner guide

Modeling Runtime

Modeling owns training, inference, tuning execution, artifact persistence helpers, runtime planning, batching, scoring, and the bridge between model outputs and objectives/evaluators.

src/spice/modeling Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

It sits after dataset builders and before prediction/evaluation/storage commits. Workflows call modeling services to train artifacts, run trials, and score saved models.

Beginner Mental Model

This is the machine-learning engine room. Dataset builders prepare tensors; model families build networks; prediction contracts define losses and decoding; modeling coordinates CUDA/runtime choices, batches, epochs, early stopping, inference, and summaries.

Input And Output Flow

  • Compile dataset-builder, model-family, prediction, and objective contracts.
  • Prepare runtime and batch plans.
  • Run epochs or inference batches.
  • Summarize metrics and persist artifact/study state through storage.

Common Confusing Concepts

Loss

A numeric penalty the optimizer tries to reduce. It converts wrong predictions into gradients.

Backpropagation

The algorithm that computes how each model weight contributed to the loss.

Runtime plan

A practical plan for device, precision, memory budget, and batching before heavy tensor work starts.

Scoring service

The bridge that turns trained model outputs into prediction metrics or evaluator summaries.
