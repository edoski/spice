Minimum Block Fee Multitask Prediction

This concrete prediction family trains the model to choose the candidate block offset with the minimum fee while also learning an auxiliary minimum log-fee regression target.

src/spice/prediction/families/min_block_fee_multitask Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

It implements the prediction-family contract consumed by modeling and evaluated later through temporal execution policies and evaluators.

Beginner Mental Model

The model gets two related lessons. One head learns "which candidate offset is best?" and another head learns "how low is the fee there?". The auxiliary task can improve shared representation learning.

Main Source Files

__init__.py
Compiles the family contract, fits training state, computes batch loss/state, and decodes candidate offsets.
batch.py
Materializes min-block-fee targets, prepared targets, and training state such as class weights.
loss.py
Computes multitask loss from offset classification and fee regression outputs.
metrics.py
Tracks classification counts, macro F1, inverse-frequency weights, loss totals, and epoch accumulators.
outputs.py
Defines output head ids, output specs, masked logits, and masked argmax decoding.

Input And Output Flow

  • Materialize targets from temporal outcome facts.
  • Build output specs for offset logits and min log fee.
  • Train with multitask loss and class weighting.
  • Decode masked offset logits into DecodedOffsets.

Common Confusing Concepts

Multitask learning

One network optimizes more than one related target. Shared layers can learn better features because they receive richer feedback.

Class weights

If some offsets are rare, inverse-frequency weights keep the model from ignoring rare classes.

Masked argmax

The model may score every offset, but invalid offsets are suppressed before choosing the best one.
