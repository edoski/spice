Prediction Families

Prediction families are concrete implementations of prediction contracts. They define target construction, output specs, loss, training-state fitting, metrics, and decoding for one task family.

src/spice/prediction/families Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

This package is the implementation namespace below generic prediction contracts. The currently documented concrete family is min_block_fee_multitask.

Beginner Mental Model

The generic prediction layer defines the interface. A family fills in the math for a specific learning problem.

Main Source Files

__init__.py
Namespace package for concrete prediction families.
min_block_fee_multitask/
Concrete family that predicts the minimum-fee block offset and a minimum log-fee auxiliary target.

Input And Output Flow

  • A registry dispatches to the selected family.
  • The family provides target materialization, output heads, loss, metrics, and decode logic.
  • Modeling calls the compiled contract without knowing family internals.

Common Confusing Concepts

Family comparison

Two prediction families can use the same model architecture but train different targets or decode different decisions.

Shared ABI

Families expose a common contract shape so the modeling loop does not need family-specific branches.
