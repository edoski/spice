Spice beginner guide

Prediction

Prediction defines what the model is trained to output, how batches and targets are represented, how losses/metrics accumulate, and how raw outputs become decoded decisions.

src/spice/prediction Docs: ARCHITECTURE.md

Where It Fits

It sits between model families and evaluation. Model families emit output heads; prediction families assign meaning, loss, metrics, and decoding rules.

Beginner Mental Model

A neural network outputs tensors. Prediction code says which tensor is an offset classifier, which tensor is a fee estimate, what the correct targets are, and how to turn tensors into an action.

Main Source Files

base.py
Defines prediction head/output specs that model families use when building output heads.
contracts.py
Defines model input batches, target batches, prepared targets, epoch accumulators, prediction batches, and compiled prediction contracts.
decoded_offsets.py
Defines the decoded-offset result id, decoded offset model, and require helper.
decoding.py
Defines generic decoded prediction results, decode input batches, action-space context, and tensor coercion helpers.
registry.py
Validates supported prediction-family ids and compiles the selected prediction contract.
__init__.py
Prediction package exports.

Input And Output Flow

  • Config selects a prediction family.
  • The contract defines targets, output heads, loss, metrics, and decoding.
  • Training uses the loss and accumulators.
  • Inference decodes model outputs into typed results.

Common Confusing Concepts

Logits

Raw scores before probabilities. Classification loss can use logits directly.

Decoded result

A typed decision after raw tensors have been converted into domain meaning.

Action masking

Some candidate offsets are illegal. Masking prevents them from winning decoding.
