Storage Catalog

The storage catalog is a searchable index of dataset, study, and artifact roots. It records enough metadata for selectors, listing, relationship queries, and refresh/reindex operations.

src/spice/storage/catalog Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

It is a subpackage of storage. It reads root-local state to build catalog records, but root-local state remains authoritative.

Beginner Mental Model

The catalog is like a library card index. It helps you find roots quickly, but the actual book is still the root directory and its state database.

Main Source Files

codecs.py
Encodes and decodes remote catalog records for transfer/sync use.
index.py
Lists and resolves dataset/study/artifact records and relationship queries.
materialization.py
Builds catalog records from root locations and validates root location facts.
records.py
Defines catalog record models for datasets, studies, and artifacts.
registry.py
Maps root kinds to catalog specs and record types.
schema.py
Defines catalog database schema.
store.py
Ensures catalog DB, upserts/deletes/lists records, and converts rows to records.
__init__.py
Catalog package boundary.

Input And Output Flow

  • A root is committed or refreshed.
  • Materialization reads root facts and builds the right record type.
  • Store upserts the record into the catalog DB.
  • Selectors and CLI list/show commands query the catalog.

Common Confusing Concepts

Refresh

Rebuild catalog rows from root-local state. This fixes stale indexes without changing the root data.

Selector

A user-facing query that narrows to exactly one catalog record or reports ambiguity.

Timestamp rule

Catalog timestamps describe index changes, not scientific meaning.
