Spice beginner guide

Storage

Storage owns durable roots for corpora, studies, and artifacts. It manages layout, ids, root-local SQLite state, payload codecs, lifecycle commits, inspection, selection, deletion, and catalog reindexing hooks.

src/spice/storage Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

It is the persistence boundary used by workflows, modeling, corpus, benchmarks, CLI storage commands, and remote transfer code.

Beginner Mental Model

Storage is the project filing system. A corpus root stores dataset state, a study root stores tuning/search state, and an artifact root stores a trained model plus summaries. The catalog is an index of those files, not the source of truth.

Input And Output Flow

  • A workflow materializes expected root facts.
  • Work happens in a staged root or existing root.
  • Payload stores write root-local state.
  • Transactions promote or mutate roots and reindex the catalog.

Common Confusing Concepts

Root

A top-level durable directory for one corpus, study, or artifact identity.

Catalog vs state DB

The root-local state DB is authoritative. The catalog is a searchable index that can be rebuilt.

Stable id

Ids come from canonical identity payloads so the same meaning maps to the same storage location.

Staged commit

New roots are built in a staging area, then promoted only after state is complete.
