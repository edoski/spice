Temporal Compilers

Temporal compilers turn problem configs into compiled problem contracts. They decide how samples, context windows, candidate offsets, and runtime metadata are materialized.

src/spice/temporal/compilers Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

They are owned by temporal, selected by config, and used during dataset preparation.

Beginner Mental Model

A compiler is like a lesson-plan builder. It takes the general question and the available rows, then writes the exact examples the model will study.

Main Source Files

base.py
Defines the base problem compiler config type.
observed_time_window.py
Concrete compiler for observed time windows, nominal/recent-median slot spacing, runtime metadata, and compiled problem stores.
registry.py
Registers problem compilers, coerces compiler configs, compiles problems, and serializes runtime metadata.
__init__.py
Compiler package boundary.

Input And Output Flow

  • Config selects a problem compiler id.
  • Registry validates the concrete config type.
  • The compiler reads feature/corpus rows and constructs a problem store.
  • Runtime metadata is saved so later artifact inference can match semantics.

Common Confusing Concepts

Observed time window

A concrete way to define candidate slots using observed block timing rather than a purely theoretical clock.

Slot spacing

The expected time between candidate action slots. It can be fixed nominally or inferred from recent blocks.

Runtime metadata

Facts discovered during compilation that must be persisted to reproduce or evaluate the problem later.
