Execution Policy

Execution policies define what a decoded model decision means in time and how to judge whether that decision was reachable, missed a deadline, or mapped to a realized block outcome.

src/spice/temporal/execution_policy Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

They sit between prediction decoding and evaluation. Prediction can choose an offset; execution policy explains how that offset behaves in the simulated temporal world.

Beginner Mental Model

If prediction says "choose action slot 3", execution policy says whether slot 3 exists, whether it arrived before the deadline, and which fee outcome it produces.

Main Source Files

base.py
Defines execution-policy config, prepared action spaces, temporal outcome facts, realized selections, decoded offsets, and compiled contracts.
strict_deadline_miss.py
Concrete policy where decisions that miss the deadline are treated strictly and overflow action slots are handled explicitly.

Execution-policy package boundary.

Input And Output Flow

  • Prepare an action space for each sample.
  • Prepare temporal outcome facts from the problem store.
  • Prediction decodes offset selections.
  • Policy realizes those selections into comparable outcome batches.

Common Confusing Concepts

Action space

The legal candidate offsets for a sample. Some offsets may be masked out.

Deadline miss

A model can choose an action that is too late. The policy defines how costly or invalid that is.

Reachable optimum

The best possible valid action under the same constraints, used as a benchmark.
