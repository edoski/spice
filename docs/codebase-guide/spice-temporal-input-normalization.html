Input Normalization

Input normalization scales feature values so training is numerically stable while avoiding leakage from future data.

src/spice/temporal/input_normalization Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

It is used by dataset builders while preparing model inputs from temporal problem stores.

Beginner Mental Model

Features can have very different numeric ranges. A neural network trains better when inputs are centered and scaled, but the scaler must be fit only from allowed training context, not future answers.

Main Source Files

base.py
Defines normalization configs, compiled contracts, local specs, config coercion, and compile dispatch.
row_standard.py
Concrete normalizer that standardizes values using row-level behavior.
scaling.py
Computes scaler stats and applies transforms to feature matrices and problem-store features.
window_weighted_standard.py
Concrete normalizer that fits standardization with window-aware weighting.
__init__.py
Input-normalization package boundary.

Input And Output Flow

  • Dataset preparation selects training calibration rows.
  • The normalizer fits mean/scale stats from allowed data.
  • Feature matrices are transformed.
  • Runtime metadata records how scaling was done.

Common Confusing Concepts

Standardization

Subtract a mean and divide by a scale so values are easier for the model to optimize.

Leakage

Using future evaluation data to fit a scaler would let the model see information it should not know.

Window-weighted

Some rows can count more when estimating scaling stats, matching the temporal window structure.
