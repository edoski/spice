Spice beginner guide

Workflows

Workflows orchestrate complete tasks: acquire a corpus, train an artifact, tune a study, or evaluate an artifact. They connect configs, storage roots, feature building, temporal compilation, modeling, prediction, and evaluation.

src/spice/workflows Docs: ARCHITECTURE.md, IMPLEMENTATIONS.md

Where It Fits

They sit above most domain services and below the CLI/remote runner. A workflow should receive a resolved config and explicit roots, then coordinate domain code without re-resolving user intent.

Beginner Mental Model

A workflow is a recipe being executed. Config says what ingredients to use; storage says where durable state lives; the workflow calls each specialist in order.

Main Source Files

acquire.py
Runs block acquisition and corpus assembly asynchronously.
evaluate.py
Loads an artifact, prepares inference samples, runs evaluator scoring, and records evaluation state.
preparation.py
Prepares root handles, validates corpus coverage, and builds prepared workflow objects before execution starts.
reporting.py
Formats workflow facts and progress callbacks for operator output.
train.py
Runs baseline or tuned training and persists the artifact root.
tune.py
Runs tuning trials and records study state.
__init__.py
Exports workflow package behavior.

Input And Output Flow

  • Prepare roots and validate coverage.
  • Compile or load the required contracts.
  • Run the domain-specific task.
  • Commit resulting corpus, study, artifact, or evaluation state.

Common Confusing Concepts

Workflow vs command

A command parses flags. A workflow performs the actual domain task.

Prepared workflow

Preparation collects validated roots and config into one object so the run function does not keep rediscovering facts.

Coverage validation

Training and evaluation require enough historical blocks before target windows; workflows check that before expensive work starts.
