Spice beginner guide

Spice Package

This is the top-level Python package. It defines shared vocabulary used across the whole project and points readers toward the major domains: config, acquisition, corpus, temporal problems, modeling, prediction, evaluation, storage, benchmarks, workflows, and CLI.

src/spice Docs: ARCHITECTURE.md

Where It Fits

The package root sits above every domain package. It should stay light: shared terms live here, while concrete behavior lives in the owning subpackage.

Beginner Mental Model

Think of this package as the table of contents plus the project dictionary. The system is a pipeline: blockchain blocks become rows, rows become features, features become temporal examples, examples become tensors, models produce predictions, predictions become fee decisions, evaluators score those decisions, and storage keeps the durable result.

Main Source Files

__init__.py
Keeps package initialization minimal and sets the matplotlib config directory environment hook used by plotting-dependent libraries.
metrics.py
Defines metric descriptors, metric sets, and window summaries so training and evaluation can describe scores consistently.
semantics.py
Defines semantic identity models for features, problems, policies, objectives, predictions, datasets, studies, and artifacts. These are stable descriptions of what something means, not how it is stored.

Input And Output Flow

  • Shared semantic models describe domain choices.
  • Domain modules compile configs into runtime contracts.
  • Workflows compose those contracts into end-to-end runs.

Common Confusing Concepts

Semantics vs config

Config is user intent and runtime setup. Semantics is the durable meaning of that setup, used to compare roots and artifacts later.
