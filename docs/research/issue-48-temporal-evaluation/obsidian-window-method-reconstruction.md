# Old fee-regime window plots and the exhaustive-replay replacement

The old figures contain three separate window experiments. They should not be treated as one protocol. The 08/06 figures are different again: they plot prediction-horizon seconds against profit, classification, and fee-regression scores. They answer how performance changes with `K`, not how testing behavior changes with fee regime. The relevant embeds resolve to `benchmarks/figures`; the later fee-regime embeds resolve to the vault's `notes/benchmark_figures` directory.

The 300/1,200 results show that duration is a smoothing choice, not an independent replication. For example, Avalanche's profit/volatility correlation changed from `r=0.219` at 1,200 blocks to `r=-0.095` at 300 blocks; the pooled median-fee correlation changed from `-0.271` to `-0.064`. See local Obsidian `TODO.md`, lines 303-335. Keeping both scales without a duration-specific thesis claim adds researcher degrees of freedom.

The Polygon “bulk” views also changed presentation after evaluation. Fee plots removed covariate outliers using an IQR rule. The cross-chain volatility plot removed profit-over-baseline outliers using a five-IQR rule, which is outcome-conditioned filtering. [`render_lstm_block_count_quartile_results.py:361`](../../../benchmarks/scripts/render_lstm_block_count_quartile_results.py#L361). Neither filter should define a final testing view.

All linked regime figures plot either legacy `profit_over_baseline` or exact-optimum hit rate. The render joins carried `cost_over_optimum` and baseline cost over optimum, but no linked scatter plotted them. [`render_lstm_wall_clock_quartile_results.py:178`](../../../benchmarks/scripts/render_lstm_wall_clock_quartile_results.py#L178). Legacy profit was the mean event fraction `(B-R)/B` under Poisson exposure, not the now-approved finite ratio `sum(B-R)/sum(B)`. ## Lean implication for exhaustive full-range replay

Representative-window selection is unnecessary once every eligible testing origin is evaluated exactly once. Its old purpose was to buy balanced regime coverage under a finite replay budget. Retaining it after exhaustive evaluation would discard data, distort testing-range prevalence, and duplicate the same outcomes through correlated 300/1,200 and fee/volatility views.

A regime figure still needs a local x descriptor. “Median base fee” and base-fee volatility are not properties of one origin until a context span is defined. Use one predeclared trailing, causal block context per origin, preferably the already-approved model/feature context rather than a new 300/1,200 evaluation duration:

- attach trailing median base fee and `std(diff(log(fee)))` to every eligible origin;
- form chain-specific deterministic bins, such as quartiles, from those descriptors;
- within each bin, add the approved raw numerators and denominators and recompute savings, opportunity, regret, safety, and action diagnostics;
- plot the bin's descriptor center on x and its finite ratio on y; show exact counts and raw totals, with no Poisson repetition, replay interval, p-value, or confidence interval.

This is descriptive stratification of the one full-range census, not window selection or repeated testing. A per-origin scatter may be added as a faint background, but binned ratio-of-sums is clearer and matches the approved accounting.

Use block context for the main regime view. The primary estimand already weights equal block opportunities, `K` is in blocks, and a fixed block context gives every descriptor the same observation count. Report its realized UTC span descriptively. Use wall-clock context only if the thesis makes a separate claim about behavior over a named elapsed-time regime.

Episode windows remain useful only for a different question: “Is performance stable across contiguous episodes of a stated length?” If that claim is wanted, partition the full testing range into every non-overlapping window of one predeclared size and aggregate the already-computed origin records. Do not select representative windows or rerun evaluation. The proposed fee/volatility figures ask the regime-conditioning question, so this extra episode view is not necessary.
