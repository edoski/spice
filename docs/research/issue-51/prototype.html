2. Turn known parent facts into inputs

Suppose the final parent row has base fee 24,000,000,000 wei/gas, gas used 27,000,000 gas, and gas limit 36,000,000 gas. The mandatory common features are:

log_base_fee_per_gas = ln(24,000,000,000 / (1 wei/gas)) = 23.901320
gas_utilization      = 27,000,000 / 36,000,000             = 0.75

Ethereum also computes the exact forming-child base fee from parent facts. Under this illustrative EIP-1559 parent state it is 25,500,000,000 wei/gas, so:

log_exact_forming_base_fee_per_gas = ln(25,500,000,000 / (1 wei/gas))
                                   = 23.961944

Polygon and Avalanche omit that column. They receive no zero, null, estimate, placeholder, or generic chain-adapter field. Any validation-selected capacity/activity or UTC-hour pair follows after these columns in the frozen order.

3. Keep inputs and outcomes on opposite sides of the decision

ValueShape/unitAvailable whenUse
Closed-row features[C,F] float32, scaledBy close of hModel input.
Future base fees[K] int64, native wei/gasOnly after h+K closesTraining/evaluation outcome, never input.
Earliest minimum labelscalar int64Only after the complete outcomeClassification target.
Minimum-fee z targetscalar float32Only after the complete outcomeAuxiliary regression target.

At a role boundary whose later role starts at block B, retain an earlier origin only when h + K < B. Training alone fits weights, input statistics, target statistics, and class supports. Validation selects. Testing reports.

4. Construct the two targets

Illustrative future base fees for h+1…h+5 are [25.5, 23, 21, 20, 22] gwei/gas. Raw integer comparison happens before floating conversion:

label k = earliest argmin([25.5, 23, 21, 20, 22]) = 3
O       = 20,000,000,000 wei/gas
ell     = ln(O / (1 wei/gas)) = 23.718998
z       = (ell - mu_training) / sigma_training

The target state is fitted once from retained training origins for the exact chain and K, using float64 population mean and standard deviation (ddof=0). No epsilon, clipping, log1p, constant-target fallback, or held-out refit exists.

5. Train and decode

MinBlockFeeOutput:
  action_logits [B,K]
  minimum_fee_z [B]

c_i = cross_entropy(action_logits_i, label_i, reduction="none")
r_i = smooth_l1_loss(minimum_fee_z_i, target_i, reduction="none")
L_total = (sum_i c_i + sum_i r_i) / N

k = argmax(action_logits, dim=-1)  # first index wins ties
target_block = h + 1 + k

If logits are [0.2, 1.1, -0.1, 1.7, 0.5], decode returns k=3 and target block 25,400,004. The auxiliary scalar remains a real training/reporting output but is not an action, fee quote, confidence value, or repair signal.

6. Evaluate economic meaning without claiming execution

B = fee at k=0 = 25.5 gwei/gas
R = fee at selected k=3 = 20 gwei/gas
O = hindsight minimum = 20 gwei/gas
S = B - R = 5.5
G = B - O = 5.5
Q = R - O = 0
assert S + Q == G

Across a complete declared range, publish sum(S)/sum(B), sum(G)/sum(B), sum(Q)/sum(B), and sum(S)/sum(G) when defined. Keep chains separate. These are finite target-base-fee-per-gas opportunity measures, not profit, transaction cost, inclusion, or actual execution.

selected_action_wait_seconds may appear only as the evaluation-owned descriptive observation field: zero for k=0, otherwise timestamp(h+k)-timestamp(h). It is not a domain/glossary term and not broadcast or receipt latency.

7. Bound HPO and preserve serving parity

HPO role

HPO receives one frozen TuneRequest, one finite typed MethodSpace, and operator-supplied complete Method values. It selects only by complete-validation total loss in current result order. It has no trial-count budget, sampler, pruning, test access, winner field, or HPO weight promotion.

Serving parity

The same artifact fixes chain, C=200, K, ordered features, scaler, target state, concrete model, and state dict. Live preparation freezes latest head h, returns [1,C,F], validates both heads, decodes k, and returns only head_block, selected_action_k, and target_block.

8. Separate paper fact from FABLE ownership

9. State limitations where the reader first needs them

  • Base fee per gas omits priority fees, transaction gas workload, inclusion, latency utility, and revenue.
  • One ML seed supports descriptive evidence, not seed robustness or a confidence interval.
  • Different K values are different class problems; testing cannot choose a best K.
  • Chains use different native assets and ranges; their totals are never pooled.
  • The auxiliary scalar is not calibrated uncertainty or an authoritative quote.
  • Historical SPICE artifacts remain valid only for their historical contracts and are not compatibility inputs.

Undergraduate reader check

  • Can a reader state what is known at h and what belongs to h+1…h+K?
  • Can they derive tensor shapes and target block for one example?
  • Can they explain training, validation, and testing ownership?
  • Can they distinguish loss, predictive diagnostics, and economics?
  • Can they state paper lineage and FABLE-owned extensions without calling FABLE SPICE?
  • Can they find one exact schema or command in under two link hops?

Source and link check

  • Every paper fact points to the paper evidence, not current code.
  • Every framework fact points to official PyTorch, Lightning, Polars, Pydantic, or Typer docs.
  • Every FABLE design fact points to the owning approved contract or realized source.
  • Relative links resolve from their final file location; anchors exist.
  • Historical SPICE paths remain verbatim and clearly nonnormative.
  • No link treats a research prototype as production API authority.

Equation check

  • Recompute EIP-1559 parent-to-forming fee using integer ordering/truncation.
  • Recompute feature logs/ratios with declared units and domains.
  • Recompute earliest minimum, target z, CE, Smooth L1, and total-loss denominator.
  • Recompute argmax tie behavior and h+1+k.
  • Recompute S, G, Q and assert S+Q=G per row and in totals.
  • Check every denominator, undefined case, dtype, reduction, and direction.

Ownership check

  • Orientation, tutorial, theory, architecture, reference, glossary, ADR, source, and research each own one kind of fact.
  • No paired taxonomy, duplicate system map, or implementation inventory remains.
  • Every local guide passes the deep-module deletion test.
  • Counts/IDs come from code or CLI, never manually synchronized prose.
  • No registry, generator, synchronizer, inventory, or absence test was added.
  • One manual final-tree review confirms the realized set.
